Introduction to HTML

1. What is HTML?

Welcome to the Byte-Sized Lesson on HTML!

HTML stands for HyperText Markup Language

What is Hypertext? It's textâ€¦ that's hyper! It's like regular text but it can do additional things, like link to other text, include images in the text, and do all of the things that you see on the Web.

And what is a markup language? It's the language that turns regular text into hypertext by marking it up with tags!

For example, here is some regular text: I'm regular text!
Here is regular text that's marked up: <b>I'm regular text!</b>
And it comes out like this: I'm regular text!

2. What is HTML Good For?

HTML is necessary for the internet. In the beginning, webpages were meant to be static pages that could be shared on the web. HTML provides all the tools necessary to make these static webpages.

However, static webpages weren't enough! People wanted webpages with content that was styled and positioned as they pleased! People wanted webpages that had functionality, that can really do things! And that's why today, along with HTML, we also have CSS and Javascript.

But, as useful as they are, CSS and Javascript are nothing without HTML, so it's important to have a good grasp of the basics before moving on to more complex webpages.

And so, let's learn a little bit about the basics.

3. Anatomy of an HTML Web Page

Take a look at this sample HTML page here:

<html>
<head>
</head>
<body>
Hello, World!
</body>
</html>

What's going on here? Well, this here is our first HTML page!

Go ahead and copy this code into a simple text document, and save that text document as myFirstPage.html. Then, double click on that text file, and voila: your browser will open it as a web page!

If it didn't work right, then you might not have the best text editor for this purpose: I would recommend you use either Notepad ++ for PC, or Smultron for PC.

4. Dissecting our HTML

Let's take another look at our HTML code:

<html>
<head>
</head>
<body>
Hello, World!
</body>
</html>

HTML is centered around the idea of tags. Tags are the things inside of brackets that you see in the code.

Everything in our example is a tag, except for one little line of text: "Hello, World!". This is the only text that appears in our webpage.

Tags remain invisible. That's because tags are just instructions that are meant for the browser, not for the viewer.

5. Tags EVERYWHERE!

Let's take another look at our HTML code:

<html>
<head>
</head>
<body>
Hello, World!
</body>
</html>

Tags generally come in pairs: an opening tag, and a closing tag. The first tag in a pair is the opening tag, and the second tag is the closing tag.

You can recognize a closing tag because it has a forward-slash. For example, <body> is an opening tag, and </body> is a closing tag.

Everything in between the opening tag and the closing tag is considered to be inside that section of the code. Therefore, "Hello, World!" is inside of the body section, and the body section is inside of the HTML section.

6. Go and see for yourself!

In fact, everything is inside the HTML section! That's because every webpage always begins with an opening <html> tag and ends with a closing </html> tag. Always.

Don't believe me? Well hey, you can check this out for yourself!

The easiest way to do this is using Google Chrome. Inside of Chrome, you can look at the code for any webpage by right-clicking anywhere on the page and selecting 'View Page Source'.

Oh, but just a word of warning: if you're new to HTML, you might be overwhelmed by what you see.

This is because an HTML page often contains inside of it confusing CSS and Javascript, and even pages that may seem simple at first (such as the Google homepage) are actually extremely sophisticated under the hood.

Webpages have become quite complex these days, but do not fear! With patience and perseverence, you'll learn how to navigate through the slush and identify the important things.
